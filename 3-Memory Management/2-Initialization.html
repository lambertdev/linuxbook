Initialization

内核的初始化过程，按照先后顺序会经历以下几个阶段：

  1. Loader跳转到内核后的实模式(Real Mode)
  2. 32位保护模式跳转到64位长模式
  3. 内核解压
  4. 页表映射建立，执行平台相关初始化代码
  5. 执行平台无关初始化代码

本章节介绍以下部分：

  • 内核虚拟地址空间的建立
  • 内核地址空间随机化(KASLR)及内核解压
  • 平台初始化过程中的内存初始化

  1. 解压前，startup_32 (0x100000)建立4GB Boot页表 (pgtable in .pgtable)，跳转到startup_64(0x100200)
  2. startup_64
    1. boot_stack_end = 0x4bb6000 (移动到LOAD_PHYSICAL_ADDR后的栈地址)
    2. trampoline 0x9d000
    3. Copy compressed kernel
      1. %rsi = 0xe34ff8
      2. %rdi = 0x4ba1ff8 (移动到LOAD_PHYSICAL_ADDR后的_bss地址)
    4. 计算.Lrelocated地址 (移动到LOAD_PHYSICAL_ADDR后的地址) 0x4b8f060
  3. 跳掉.Lrelocated地址执行，加载第二阶段IDT，boot_page_fault->do_boot_page_fault
    1. 主要作用是add_identity_map 到top_level_pgt
  4. 创建 identity maps
    1. [_head, _end]
    2. [boot_params]
    3. [cmdline]
    4. 更新CR3位top_level_pgt
  5. B在0x0000000004b8f0b6 是解压内核后的返回处
  6. 更换新页表在0x100007d
